## Порядок действий

### Шаг 1. Вынести общую логику OpenRouter

**Файл:** `lib/openrouter.ts`

**Что сделать:**
- Оставить единую функцию `chatCompletion(systemPrompt, userPrompt)` и модель `openrouter/owl-alpha`.
- Вынести формирование user-prompt из статьи в общий хелпер `buildArticlePrompt(article)` (заголовок, дата, текст, обрезка до `MAX_CONTENT_CHARS`).
- Использовать `lib/env.ts` для ключа и `HTTP-Referer` (локально и на Vercel).

**Зачем:** три новые кнопки отличаются только system-prompt; парсинг и вызов API — общие.

---

### Шаг 2. Добавить три AI-функции с промптами

**Файл:** `lib/openrouter.ts` (или `lib/prompts.ts` + импорт в openrouter)

| Action | Функция | System-prompt (суть) | Формат ответа |
|--------|---------|----------------------|---------------|
| `summary` | `summarizeArticle()` | Кратко объяснить, о чём статья, на русском | 2–4 абзаца, без воды |
| `theses` | `extractTheses()` | Выделить ключевые тезисы на русском | Маркированный список, 5–10 пунктов |
| `telegram` | `generateTelegramPost()` | Пост для Telegram-канала на русском | Цепляющий заголовок, 2–3 абзаца, ссылка на источник, умеренно эмодзи |

**Для каждой функции:**
1. Проверить, что есть `title` или `content`.
2. Собрать prompt через `buildArticlePrompt(article)`.
3. Вызвать `chatCompletion` с нужным system-prompt.
4. Вернуть строку `text`.

**Промпты уточнить:**
- Язык ответа: **русский** (как у «Перевод»).
- Не выдумывать факты — только на основе текста статьи.
- Для Telegram: в user-prompt передать исходный URL статьи для вставки в пост.

---

### Шаг 3. Подключить действия в API

**Файл:** `app/api/analyze/route.ts`

**Что сделать:**
1. После `fetchAndParseArticle(url)` разветвить по `action`:

```
summary   → summarizeArticle(article)
theses    → extractTheses(article)
telegram  → generateTelegramPost(article, url)
translate → translateArticle(article)   // уже есть
```

2. Для всех AI-действий возвращать единый формат:

```json
{ "text": "..." }
```

3. Убрать возврат сырого JSON парсинга для `summary` / `theses` / `telegram` — пользователь ждёт сгенерированный ответ, не структуру парсера.

4. Сохранить `maxDuration = 300` (перевод и генерация могут занимать до ~1 мин).

**Обработка ошибок:**
- 422 — не удалось распарсить статью.
- 500 — ошибка OpenRouter / таймаут.
- Понятные сообщения на русском в `{ error }`.

### Шаг 4. Обновить интерфейс

**Файл:** `components/ArticleAnalyzer.tsx`

**Что сделать:**
1. Для `summary`, `theses`, `telegram` показывать `data.text` как **plain text** (как у «Перевод»), не JSON.
2. Обновить подписи загрузки:
   - «О чем статья?» → «Генерация краткого содержания…»
   - «Тезисы» → «Выделение тезисов…»
   - «Пост для Telegram» → «Создание поста…»
3. Флаг `isPlainText`: `true` для всех четырёх AI-кнопок (`summary`, `theses`, `telegram`, `translate`).

**Опционально (позже):**
- Кнопка «Скопировать» в блоке результата.
- Для Telegram — предпросмотр с моноширинным форматированием ссылок.

### Шаг 5. Проверка и тестирование

**Локально:**
```powershell
npm run dev
```

| Кнопка | URL для проверки | Ожидание |
|--------|------------------|----------|
| О чем статья? | Любая англоязычная статья с `<article>` | Краткое содержание на русском |
| Тезисы | Та же | Список тезисов на русском |
| Пост для Telegram | Та же | Готовый пост со ссылкой |

**На Vercel:**
- `OPENROUTER_API_KEY` в Environment Variables.
- Redeploy после изменения переменных.
- Учесть лимит timeout на Hobby (10 с) — для длинных запросов нужен Pro.

**Сборка:**
```powershell
npm run build
```

## Порядок реализации (кратко)

```
1. Рефакторинг openrouter.ts (общий prompt + chatCompletion)
        ↓
2. summarizeArticle / extractTheses / generateTelegramPost
        ↓

        ↓
4. UI: plain text + подписи загрузки для трёх кнопок
        ↓
5. Ручное тестирование локально и на Vercel
```

- Шаг 5 — финальный, после 1–4.
